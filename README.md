# Post-War Structural Damage Assessment & Recovery Expert System

**Author:** Dana Nidal

---

## Project Overview
This project implements a **Rule-Based Expert System** designed to handle complex decision-making during post-war engineering recovery and structural damage assessments. Evaluating heavily impacted urban structures involves identifying not only visible architectural damage but also systemic data risks, overlapping bureaucratic errors, and physical landscape hazards.

By employing advanced logical inference mechanisms—**Forward Chaining** and **Backward Chaining**—this expert system flags architectural dangers and alerts recovery coordinators to administrative errors and systemic database anomalies.

## Core Mechanisms Implemented

### 1. Forward Chaining (Data-Driven Inference)
The forward-chaining engine evaluates incoming baseline facts to diagnose structural vulnerabilities and generate warnings. Key warning paradigms include:
* **Ghost Database Problem:** Identifies coordination risks where data is missing, conflicting, or uncoordinated.
* **Overlapping Echoes:** Flags redundancies or conflicting statements within legal and structural compensation claims.
* **Shifting Sands:** Detects severe structural/geological hazards, such as building terrain instability or the presence of explosive conflict debris.
* **Superficial Assessments:** Minimizes risk by flagging when structural checks are deemed rushed or inadequate.

### 2. Backward Chaining (Goal-Driven Inference)
The backward-chaining engine starts with a target hypothesis or risk scenario (e.g., *Is the 'Ghost Database Problem' present?*) and traces back through the logical rules to see if the available field metrics support that conclusion.

## Libraries & Tools Used
* **`experta`**: A specialized Python framework used to build production-grade, rule-based expert systems inspired by CLIPS.
* **Python 3**: The core development language.
* **Jupyter Notebook**: The execution environment for writing rules, handling logic tests, and viewing output data.
